# gorderly v0.3.2

## Docs

Added `docs/FRAMEWORK.md`: how we write tests with `spec` + `expect` --
structural functions and lifecycle hooks, context nesting, the `subject`
pattern, mocking/stubbing, and why `spec` rather than Ginkgo. The
README's old "Writing tests with spec" and "full toolchain" sections
(which carried the same content inline) now point here instead, so it
isn't duplicated in both places. Mirrors
[`xctidy`'s own `docs/FRAMEWORK.md`](https://github.com/woodie/xctidy/blob/main/docs/FRAMEWORK.md)
on the Swift side.

Also documents a real `go test` limitation found while reviewing issue
#1: a literal `/` in a `describe`/`context`/`it` string is
indistinguishable from a `t.Run` nesting boundary once it reaches `go
test -v`'s output -- baked into the `testing` package itself, not
something `gorderly` can parse around. The fix is on the spec-author
side (substitute a lookalike character, as `expect`'s own
`expect_test.go` already does); see the README's Limitations list and
`docs/FRAMEWORK.md`.

## Fix

The `Makefile`'s `check` target was defined twice -- the second
definition silently overrode the first, well-behaved one (terse: silent
on success, full log on failure) with a plain `golangci-lint run && go
test ./...` that didn't suppress anything. Removed the stray duplicate.
Also added `build`/`install` targets (`install` depends on `build`),
matching `xctidy`'s Makefile shape. Dev-tooling only -- nothing about the
released `gorderly` binary itself changed.

## Chore

`.gitignore` now excludes `.DS_Store`.
