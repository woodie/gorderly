# gorderly v0.1.0

## Initial release

`gorderly` renders plain `go test -v` output as a nested RSpec `-fd`-style tree, using the `/`-joined hierarchy `t.Run` subtests already carry in their names -- no BDD framework, no `--json-report` round trip, no wrapped test-runner binary. The same relationship `xctidy` has to `xcodebuild`, applied to `go test`.

## Usage

Pipe `go test -v`'s output straight in, or let `gorderly` run `go test` for you:

```
go test -v ./... | gorderly -fd
gorderly -fd .
```

## Styles

Three render styles, matching `xctidy`'s exact flag surface: default (classic glyph + per-test elapsed time), `-fd` (RSpec's documentation format), `-fs` (Mocha/Jest's spec format) -- `--format documentation`/`--format spec` are the long forms of the last two. All three share one closing footer, not stacked per-style summaries.

## Scope

This first release covers the core parse/render pipeline and both input modes (pipe and wrapper). Not included yet: a `go test -json` input mode, and any dots/progress format -- see the README's "Why not gotestsum?" section for why the latter is deliberately out of scope.
