# gorderly v0.3.0

## Parity fixes against `xctidy`

`gorderly` and [`xctidy`](https://github.com/woodie/xctidy) are meant to be
the same RSpec-flavored experience, ported to Go and Swift respectively. A
line-by-line comparison against `xctidy`'s `Engine.swift` turned up six
real divergences, all fixed in this release:

- **Classic style now includes the `(FAILED - N)` cross-reference** on
  failing leaves, pointing at the matching numbered entry in the
  `Failures:` section below -- it was silently missing before.
- **Classic style now colors only the glyph and the elapsed-time number**
  on every leaf (pass, fail, and skip), leaving the test name in the
  terminal's default color -- previously only passing leaves got this
  treatment; fail/skip lines were colored solid red/cyan end to end.
- **`-fd`'s pending label is now yellow**, matching real RSpec `-fd`
  convention (it was cyan).
- **`-fs`'s fail/skip leaves now match Mocha/Jest's own glyphs and text**:
  `✗ name (FAILED - 1)` and `- name (SKIPPED)`, instead of a different
  glyph with no cross-reference/marker text.
- **Added `--version`/`-v`**, checked before reading stdin so it never
  hangs waiting on piped input that isn't coming.
- **Color now auto-disables when stdout isn't a real terminal** (piped to
  a file or another process), not just when `NO_COLOR` is set.

## Usage

```
gorderly --version
gorderly -v
```

## Notes

`gorderly`'s version is a hand-bumped constant (`version.go`), not derived
from `git describe` at build time -- its primary install path, `go install
github.com/woodie/gorderly@latest`, is a module-proxy fetch with no `.git`
metadata to describe.

No behavior change to `Parse`; this release is entirely about `Render`'s
output and the CLI surface in `main.go`.
