# gorderly v0.3.1

## Fix

`--version`/`-v` didn't check `fmt.Fprintln`'s return value, tripping
`golangci-lint`'s `errcheck` (`make lint`/`make check` failed on this
alone). Matches every other write in `main.go`, all of which already use
`_, _ = fmt.Fprintf(...)`.

No behavior change -- v0.3.0's six parity fixes against `xctidy` (see
`docs/releases/v0.3.0.md`) are unaffected.
