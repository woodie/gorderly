# gorderly v0.2.0

## `-fv`: Vitest's own tree, for `go test`

A fourth render style, alongside the classic default, `-fd` (RSpec), and
`-fs` (Mocha/Jest): `-fv` (long form `--format vitest`) renders
[Vitest](https://vitest.dev)'s own terminal conventions -- `âś“`/`Ă—`/`â†“`
glyphs, a two-toned green duration (plain green number, lighter
`brightGreen` unit), and a `Test Files`/`Tests`/`Duration` footer with
labels right-justified to 11 columns -- pulled from Vitest's actual
reporter source, not guessed from its docs.

```
go test -v ./... | gorderly -fv
```

Built for monorepos running both `vitest run` and `go test` side by side
(a JS frontend next to a Go backend, for example) that want the two
outputs to read as one consistent terminal rather than two visibly
different tools bolted together -- without gorderly ever touching,
wrapping, or reformatting Vitest's own output to get there.

## Known limitation

`go test -v` only reports elapsed time to two decimal places of a second
(e.g. `0.00s`), so fast subtests will often show a flat `0ms` where
Vitest's own finer-grained JS timers would show `2ms`/`4ms`. This is a
ceiling in `go test`'s own output, not something `-fv` can recover.

## Scope

`-fv` intentionally has no `Test Files` line if ever ported to a sibling
tool with ambiguous suite nesting (see `xctidy`'s own `-fv`) -- gorderly's
version has no such issue, since one `PackageResult` per Go package was
already unambiguous.
